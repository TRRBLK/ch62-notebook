##BASE DE DATOS
Estructuras en las que se almacenan conjuntos de datos para su posterior 
procesamiento: consulta, análisis, relacionamiento o transmisión.

Una analogia a las BBDD son las bibiliotecas, donde el contenido se puede 
consultar por etiquetas.

Ejemplos son Guias telefonicas, redes sociales y transacciones bancarias.

##TIPO DE BASE DATOS
#SEGUN SU VARIEDAD
- Estáticas: solo se permite la lectura. Almacenan datos historicos para 
estudiar su comportamiento a lo largo del tiempo y realizar previsiones a 
futuro.

- Dinámicas: actualizan, reorganizan, añaden y eliminan información a lo 
largo del tiempo.

#SEGUN SU CONTENIDO
- Bibliograficas: recopilaciones cientifico-tecnico que reunen datos sobre 
un área de conocimiento.

- Directorios: listado de datos personales en formato electronico.

- De texto completo: recogen articulos completos, y cuya busqueda puede 
realizarse por palabras clave, fechas, etc.

- Especializadas: cubre necesidades puntuales de un publico objetivo que 
consume dicha informacion. 

#MODELOS DE BASE DE DATOS
Modelo relacional: datos relacionales, consiste en un esquema de tablas con 
filas y columnas (SQL)

Modelo jerarquico: parecida a un árbol, padre e hijo, compuesta por una 
colección de registros.

Modelo multidimensional.

Deductivo. Reglas y hechos almacenados.

Modelo OOP.

Modelo transaccionales. 

Data Base Management Systems.

#BBDD RELACIONAL
Permite interactuar con los datos mediante tablas para generar información 
adicional o volver a montar los datos de distintas maneras.


#SQL 
Structured Query Language. Lenguaje de programación diseñado para gestionar 
datos almacenados en BBDD relacionales.

Funciones:

- Creación de BBDD
- Nuevas tablas
- Procedimientos almacenados en la BBDD
- Vistas de una BBDD
- Asignación de permisos en tablas, procedimientos y vistas
- Consultas contra una BBDD
- Recuperación de datos
- Inserción de registros
- Actualización de registros
- Borrado de registros 

Opera a traves de declaraciónes simples y complejas, Es un lenguaje facil, 
universal, estable, de uso transversal y flexible al acceder datos.

Es una buena oportunidad laboral para los que se conviertan en expertos.

#Data Definition Language
Crear y modificar la estructura de una BBDD. Sintaxis de programación

#Data Control Language
Crear roles, permisos e integridad referencial, control al acceso a la BD.

#Data Modification Language
Permite interactuar con los datos de la BD. 	

#Transaction Control Language
Administra diferentes transacciones (conjunto de ordenes que se ejecutan de 
forma indivisible) que ocurren dentro de una BD. 

#Elementos de SQL
COMANDOS
DEFINICION
crear, modificar, eliminar
MANIPULACION
consultas, seleccionar, insertar, actualizar, eliminar registros de tabla

CLAUSURA
seleccionar datos con los que interactuar

OPERADORES
combinaciones de caracteres para realizar comparaciones y asignaciones entre 
datos. 

FUNCIONES DE AGREGADO
comando SELECT. Devolver determinado valor como promedio, redondeo, valor más 
bajo. 

CREATE (junto a PK) crear tablas, campos e indices.

ALTER agregar, eliminar o modificar campos en una tabla existente.

DROP eliminar tablas e indices

TRUNCATE eliminar todos los registros de una tabla

son comandos de DDL

FROM posicionarse en la tabla donde se van a seleccionar los registros

WHERE condiciones que deben cumplir los datos que se van a seleccionar

ORDER BY organizar la seleccion de acuerdo a un orden especifico

DISTINCT seleccionar valores unicos dentro de una columna

LIMIT limitar numero de filas en consultas

Son comandos de DML
